# SakuraMapper

[English](README_EN.md)

---

Серверный PE маппер. Сервер делает всё — парсит PE, маппит секции, применяет релокации под адрес который выдал клиент, зачищает `.reloc` и отправляет зашифрованный образ по WebSocket. Клиент просто пишет его в целевой процесс и запускает.

Смысл в том, что если кто-то сдампит регион памяти — релокационных данных там не будет. Образ уже был релоцирован на нужный адрес до того как покинул сервер.

---

## Лицензии

Ключи хранятся в `keys.json`. Два типа:

- `initial` — ключ выдан, но ещё не использован. При первом коннекте сервер считывает HWID клиента и навсегда привязывает ключ к этой машине
- `bound` — ключ привязан, попытка зайти с другой машины отклоняется

HWID собирается из CPU ID, серийника диска C: и имени компьютера, хешируется FNV-1a.

---

## Сборка

Visual Studio 2022, Windows SDK, x64.

Порядок сборки: `payload` → `server` → `client`. Всё в `SakuraMapper.sln`.

---

## Запуск

```
server.exe --dll payload.dll --port 9150
client.exe --host 127.0.0.1 --port 9150 --key SAKURA-TEST-KEY-0001 --target notepad.exe
```

Если не передать `--key` и `--target`, клиент спросит интерактивно. При первом запуске сервера без `keys.json` автоматически создаётся тестовый ключ `SAKURA-TEST-KEY-0001`.

---

Требует прав администратора. Только x64.
